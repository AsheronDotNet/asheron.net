arrow_drop_down Major Features
Nearly full (>80%) utilization of portal.dat formats, including: graphics objects, model setup objects, animations sequences, animation motion tables, region descriptor, dungeon environments, clothing tables, experience table, skill table, attribute table, and more.
Uses the complete data available from the cell.dat, including: dungeon cells and landblock/landcells.
Full and stable network system
Full physics and collision system.
Full server-driven/predicted animation system.
Landblock object distribution system.
Starter systems for inventory, experience, combat, death, and more.
Monster artificial intelligence.
Spell projectiles.
arrow_drop_down General Features
Ability to pick up, drop, move, and equip items.
Support for spending available XP on attributes and skills.
Basic logic for spawning corpses.
Basic vital (health, stamina, mana) regeneration system.
Functional lifestones.
Functional doors.
Capable of visually cloning objects that existed on live server (including players, NPC's, monsters, and others.)
Capable of spawning visually identical persistent objects for doors, NPC's, etc. using canned community data.
Loads of commands for exploring the map and experimenting with the game.
